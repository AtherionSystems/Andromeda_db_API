OKE Ops Guide

mx-queretaro-1 compartment: reacttodo cluster: andromedaoke scripts/andromeda-bringup.sh · scripts/andromeda-undeploy.sh
Cluster is spending tokens with no useful work. Stop the bleeding before anything else.
Immediate actions
01
Delete the Load Balancer highest priority
Networking → Load Balancers (region: Mexico Central Querétaro). Find the LB tied to the cluster. Click ··· → Terminate.
Expected
LB disappears within 1–2 min. Cost line "Load Balancer" drops to 0.
  • Stuck in "Deleting": LB is still referenced by a K8s Service. Run kubectl delete svc --all -n andromeda first, then retry.
02
Set node pool size to 0 first
Containers → Clusters → andromedaoke → Node Pools → pool1 → Edit → Number of nodes: 0 → Save.

Do this before terminating instances — otherwise OKE auto-heals and keeps spawning new ones.
Expected
"No nodes are associated with this node pool." The loop stops.
03
Terminate all compute instances
Compute → Instances → select all OKE nodes → Actions → Terminate. Check "Permanently delete the attached boot volume" for each.
Expected
All instances move to Terminated. No new ones spawn because pool is at 0.
  • Still spawning after step 02: delete the entire cluster — Containers → Clusters → andromedaoke → Delete. This is the nuclear option and always works.
04
Delete orphaned boot volumes
Block Storage → Boot Volumes → filter compartment reacttodo. Delete any volume not attached to a running instance.
After these steps the only remaining cost is the OKE control plane (~$0.10/hr) and Autonomous DB (near-zero at low load). The $3–4/hr Load Balancer line will be gone.
Run at end of every work session. Use the script — it handles everything automatically.
Preferred — script

Run the undeploy script from Cloud Shell
cd ~/reacttodo/oci_devops_project/MtdrSpring bash scripts/andromeda-undeploy.sh
The script will prompt for cluster OCID and node pool OCID, then handle everything: K8s service deletion, node pool scale-down, cluster deletion, and boot volume check.
Expected
Final output: "Andromeda is DOWN · Token spend stopped."
Manual — step by step
01
Delete K8s services to release Load Balancer
kubectl delete svc --all -n andromeda kubectl delete ingress --all -n andromeda
Expected
service "andromeda-service" deleted. This triggers OCI to release the LB.
  • Namespace not found: skip this step.
  • kubectl not configured: run oci ce cluster create-kubeconfig --cluster-id <ocid> --region mx-queretaro-1 first.
02
Delete the cluster entirely
Containers → Clusters → andromedaoke → (top right) Delete cluster → confirm.

Or via CLI:
oci ce cluster delete --cluster-id <cluster-ocid> --region mx-queretaro-1 --force
Expected
Cluster disappears. All node instances terminate. LB is released. Autonomous DB untouched.
03
Delete orphaned boot volumes
Block Storage → Boot Volumes → compartment reacttodo → delete any AVAILABLE volume with no attached instance.
04
Verify in Cost Analysis (10 min later)
Billing → Cost Analysis. OKE bar gone. Load Balancer = 0. Only Autonomous DB remains.
