Vault & Secrets
Schedule the Vault for deletion. OCI enforces a minimum 7-day waiting period — you cannot bypass this, but it won't block the rest of the reset.
DESTROY
OCI Vaults cannot be immediately deleted — they go into a "Pending Deletion" state for a minimum of 7 days. This is a security feature. You can proceed with the rest of the reset immediately after scheduling deletion.
01
Navigate to the Vault OCI Console
Security → Vault → compartment: reacttodo → click reactvault
02
Delete the secret inside first OCI Console
Inside reactvault → Secrets (left menu) → github_secret_token
Click github_secret_token → Schedule deletion → set pending period to 7 days (minimum) → Schedule Deletion.
Expected
Secret state changes to: Pending Deletion Scheduled deletion date shown
03
Schedule Vault deletion OCI Console
Security → Vault → reactvault → Schedule Deletion button
Click Schedule Deletion at the top of the vault page. Set pending period to 7 days. Confirm.
Expected
Vault state: Pending Deletion Vault will be permanently deleted in 7 days automatically

Wipe the home directory Cloud Shell
cd ~ rm -rf reacttodo/ 2>/dev/null rm -rf .kube/ .oci/ .config/ .ssh/ 2>/dev/null rm -f wallet.zip 2>/dev/null rm -rf wallet/ 2>/dev/null rm -rf oci_devops_project/ 2>/dev/null echo "Done. Current contents:" ls -la ~

Done. Current contents: total 0 drwxr-xr-x . drwxr-xr-x .. (only hidden system files — .bash_history, .bashrc etc)

Create the new cluster and run the bringup script. This is the same flow as the daily bring-up, but it's your first time so all secrets will be created fresh.
SETUP
01
Quick Create OKE Cluster OCI Console
Containers → Kubernetes Clusters (OKE) → Create cluster → Quick Create
Use these exact settings:
Name: andromedaoke Compartment: reacttodo Kubernetes version: v1.34.2 Shape: VM.Standard.E3.Flex Number of OCPUs: 1 Amount of memory (GB): 6 Node count: 1 Public API endpoint: checked (Public)
Expected — after ~12 min
Cluster "andromedaoke" State: Active Node pool pool1: 1 worker node Running
While waiting, copy the Cluster OCID from the Details tab — you'll need it in 2 minutes.
02
Add port 80 ingress rule to the LB security list OCI Console
Networking → VCNs → new oke-vcn-quick-andromedaoke-* → Security Lists → oke-svclbseclist-*
Add Ingress Rule:
Stateless: No
Source CIDR: 0.0.0.0/0
Protocol: TCP
Destination Port Range: 80

Do this now before the bringup script runs — otherwise the LB will be inaccessible.
Expected
Ingress Rules list now shows: TCP 0.0.0.0/0 All 80 TCP traffic for ports: 80
03
Run the bringup script Cloud Shell
Cloud Shell — from inside your repo root
cd ~/Andromeda_db_API bash scripts/andromeda-bringup.sh
The script will prompt you for:
Cluster OCID: → paste from OCI Console Details tab OCIR username: → axieboiigznv/a01571222@tec.mx OCIR Auth Token: → the token you saved in P7 DB_USERNAME: → your ADB username DB_PASSWORD: → your ADB password WALLET_TRUSTSTORE_PASSWORD: → the password you set when downloading the wallet (P8 step 09) WALLET_KEYSTORE_PASSWORD: → same password as above TELEGRAM_BOT_TOKEN: → your bot token from BotFather TELEGRAM_BOT_USERNAME: → your bot username Wallet directory: → /home/a01571222/wallet
Expected final output
======================================== Andromeda is UP External IP : 1xx.xx.xxx.xxx API base URL: http://1xx.xx.xxx.xxx/api Health check: http://1xx.xx.xxx.xxx/actuator/health ======================================== Post this IP in your team channel now.
Pod CrashLoopBackOff: wallet password wrong — delete andromeda-secrets secret and re-run script.
ImagePullBackOff: auth token wrong or expired — delete ocir-secret and re-run script.
Node not Ready after 5 min: check OCI Console → Compute → Instances — node may still be provisioning.

Verify health endpoint Cloud Shell
curl http://<EXTERNAL_IP>/actuator/health
Expected
{"status":"UP"}
{"status":"DOWN"}: DB connection failing — wallet password mismatch. Check kubectl logs.
